# Anti-AI-Slop Design Guide

How to tell when an interface was generated by an LLM instead of designed by a person — and the
rules this project follows to avoid it. Written while restyling the shipment-tracking template to
the **Harbor** palette; the before/after examples are real diffs from this repo.

> "AI slop" is the instantly-recognizable look of an interface that came out of a model with no
> designer in the loop: a purple gradient, Inter everywhere, three icon-topped cards in a row, and
> a timid palette — the same 2020-era layout repeated like a cargo-cult reenactment of "modern web
> design." The fix is never a better prompt; it's a point of view.

---

## 1. Why models converge on the same look

LLMs don't have taste — taste is lived experience compressed into an internal model of "good" that's
easy to recognize and hard to articulate. A model only has **statistical correlations between
tokens**. So it reproduces the *most common* pattern in its training data and calls it "best."

Two accidents of history set the defaults everyone now recognizes as "AI-made":

- **The purple/indigo default.** Tailwind shipped `indigo-500` as the accent in its early demo
  components. That choice got copied across five years of tutorials, component libraries, and
  landing pages, so `indigo`/`violet` is wildly over-represented in the corpus. Ask a model for "a
  modern site" with no color direction and it reaches for blue-purple every time.
- **The Inter monoculture.** Inter is the default font in nearly every AI design tool, component
  library, and site builder. The result is an aesthetic monoculture: five different SaaS pages that
  look like siblings.

When thousands of people use the same tools with the same prompts, you get thousands of near-identical
designs. A fintech homepage that could equally be a CRM or a project tool communicates nothing about
the product or the people behind it. **An ugly page with a clear point of view is more memorable than
a generic page with none.**

## 2. The tells — signal → why it reads as AI → the human rule

| Tell (signal) | Why it reads as AI | The human rule |
|---|---|---|
| **Blue/purple primary** (`indigo`, `violet`, `#6366f1`, purple-to-blue gradients) | The Tailwind-default cargo cult; "we used AI" color scheme | Pick ONE confident, on-domain hue and commit. Logistics → petrol teal, not indigo. |
| **Inter for everything** (body *and* display the same) | The default-font monoculture | Pair a distinct display face with the body font. Here: Geist (headings) + Inter (body) + DM Mono (numerals). |
| **Colored glows / big soft shadows** (`shadow-2xl`, radial glow `::before`, neon box-shadows) | Decoration with no function; "floating cards in a void" | Shadow = elevation, used only on genuinely floating layers (menu/dialog/popover). Surfaces are flat with hairline borders. |
| **Uniform radius + pill everything** (every element `rounded-2xl` or `rounded-full`) | One slider dragged across the whole UI | Geometry is a decision per element class. Buttons crisp `rounded-md`; status chips can be pills; cards a notch softer. |
| **Soft shadow on every card** | "Material-ish" default with no hierarchy | Hierarchy comes from tonal surface + border, not a drop shadow on everything. |
| **Colored single-side border stripe** (3–4px accent bar on the left/top of cards, nav items, alerts) | The signature "component-library demo" tell — decoration masquerading as an indicator | Carry state/category with a **solid tonal tint fill** or the text/icon color. No accent stripe. |
| **Rainbow charts** (5 unrelated hues: blue/cyan/lime/magenta/orange) | The d3/chart.js default palette, untouched | A mono ramp in the brand hue + ONE accent. Semantic series (delayed/exception/delivered) keep amber/red/green. |
| **Centered hero, badge above the H1** | The single most-copied landing layout | Vary the hero. Asymmetry, a real screenshot, or a dense data-first header reads designed. |
| **Identical icon-on-top feature cards, ×3** | The "three boxes with icons" tutorial grid | Differentiate cards by content weight; vary icon placement; commit to one layout instead of three variants. |
| **Numbered 1-2-3 step rows / stat-banner rows** | Filler structure that says nothing | Integrate stats into narrative; use real progression markers tied to the domain. |
| **Emoji as nav/section icons** | The fastest "vibe-coded" tell | A single icon set (here: Lucide). No emoji in chrome. |
| **Glassmorphism content cards** (`backdrop-blur` everywhere) | 2021 trend the model never moved past | Solid tonal surfaces. (A frosted *topbar* is fine — it's chrome, not content.) |
| **Vague aspirational copy** ("Build the future of work", "all-in-one platform", "best-in-class") | Generic superlatives with no specifics | Write in a real voice with specifics: "Financial infrastructure for the internet." |
| **All-caps labels everywhere** + **faint grey body text** (`text-gray-400` on white) | Default micro-typography; fails WCAG AA | All-caps only where it earns it (table heads). Body text ≥ 4.5:1 contrast. |
| **Too many font weights** (300/400/500/700/800 mixed) | No weight system | Cap at three: 400 body, 500 labels, 600 headings. |

## 3. The positive standard (what this repo does instead)

The point isn't a checklist of "don'ts" — it's having a **system with an opinion**. Harbor is that
opinion for this template.

### Color — one hue, committed
- **Petrol teal** primary (`oklch(0.515 0.082 184)`) on a **warm off-white** canvas
  (`oklch(0.985 0.004 95)`) — warm neutrals are something a model almost never reaches for (it
  defaults to cool grey). Amber (`--accent-brand`) is a *sparing* second accent, not a co-star.
- Status stays semantic: green / amber / red, tuned to the palette.
- Charts: a teal → sage → sand mono ramp + one amber accent. No rainbow. (`--chart-1..5`)
- All color flows through semantic tokens (`bg-primary`, `text-muted-foreground`, `text-success`),
  never hardcoded hex — except chart *series* colors, which are the one sanctioned exception.

### Typography — a real type system
- **Families:** Geist (display/headings) · Inter (body/UI) · DM Mono (numerals/tables).
- **Three weights only:** `400` regular (body) · `500` medium (labels, nav, buttons, table heads) ·
  `600` semibold (titles, KPI numbers). No `700` in chrome.
- **Modular scale, 14px body** (dense dashboard):

  | Role | Size | Weight | Tracking |
  |---|---|---|---|
  | Page title | 24px | 600 | −0.02em |
  | Section head | 18px | 600 | −0.015em |
  | Card title | 15px | 600 | −0.01em |
  | Body | 14px | 400 | −0.006em |
  | Meta / secondary | 13px | 400 | 0 |
  | Micro-label / table head | 12px | 500 | +0.05em, UPPERCASE |
  | KPI figure | 30px | 600 | −0.02em, `tabular-nums` |

- Negative tracking on large text, slightly positive on small caps. `tabular-nums` on every metric so
  columns align. Hierarchy from **size + weight + color**, not size alone.

### Surface & geometry
- Flat tonal cards, hairline borders, `--radius: 0.625rem`. Shadow reserved for floating layers.
- Tabs use an underline indicator (not a pill row). Sidebar active = a **solid tonal tint fill** —
  no colored left/side stripe (a side-border stripe is itself a tell; the fill carries the state).

## 5. Detection checklist

Run these greps before shipping any UI. A hit isn't automatically wrong — it's a prompt to justify
the choice.

```bash
# purple/indigo primary + gradients
grep -rniE 'indigo|violet|#6366f1|#a855f7|#8b5cf6|from-purple|via-indigo|to-violet|bg-gradient' app --include='*.vue' --include='*.css' --include='*.ts' | grep -v 'components/ui/'

# Inter used as the display font too (monoculture)
grep -nE -- '--font-display' app/assets/css/*.css   # must NOT equal --font-sans

# rainbow chart arrays (5 unrelated hues)
grep -rnE "#(3b82f6|06b6d4|a855f7|f59e0b|eab308|ec4899|f43f5e)" app/mocks app/components --include='*.ts'

# glows / oversized shadows / glassmorphism on content
grep -rnE 'shadow-(xl|2xl)|drop-shadow-2xl|backdrop-blur' app --include='*.vue' | grep -v 'components/ui/'

# weight soup (more than 3) and faint grey body text
grep -rohE 'font-(thin|extralight|light|normal|medium|semibold|bold|extrabold|black)' app --include='*.vue' | sort | uniq -c
grep -rnE 'text-(gray|slate|zinc|neutral)-(300|400)' app --include='*.vue'   # likely fails AA on white

# emoji in templates (nav/section icons)
grep -rnP '[\x{1F300}-\x{1FAFF}\x{2600}-\x{27BF}]' app/pages app/components --include='*.vue'
```
